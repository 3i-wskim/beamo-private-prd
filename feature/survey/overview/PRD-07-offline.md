## 📤 업로드 큐

### 큐 대상
오프라인 중 생성된: 태그 · 코멘트 · 폴더 · BeamoBox 파일 · 스레드 답글 · 서베이 포인트 · 서베이 컨테이너(오프라인 생성 시)

### 동작 흐름

```
생성 → 큐 저장 + 시트 즉시 표시 (업로드 대기 인디케이터)
         │
         ▼ 네트워크 복구
큐 자동 처리 (FIFO — 먼저 생성, 먼저 업로드)
         │
         ▼ 각 아이템
   Pending → Uploading → Succeeded ✅ / Failed ❌
```

### 아이템별 상태

| 상태 | UI | 사용자 액션 |
|------|-----|-----------|
| ⏳ **Pending** | 클라우드-싱크 아이콘 | — |
| 🔄 **Uploading** | 애니메이션 프로그레스 | — |
| ❌ **Failed** | Retry + Delete 버튼 | Retry → 큐 끝에 재추가 / Delete → 컨펌 → 로컬 삭제 |
| ✅ **Succeeded** | 정상 클라우드 아이템 전환 | 점 세 개 메뉴 복원 |

### 실패 처리 정책
- **Retry** → 큐 끝에 다시 추가 → 업로드 시작
- **Delete** → 컨펌 다이얼로그 → 큐 제거 + 로컬 삭제 (서버 업로드 안 됨)
- 📌 한 아이템 실패는 다른 아이템을 **차단하지 않음** — 다음 아이템 진행

### 큐 유지 정책
- **큐 최대 크기 제한 없음** — FIFO 순차 처리라 서버 부하 문제 없음
- **앱 재시작** → 큐 데이터 유지, 업로드 프로세스 자동 재개 (현행)
- **앱 삭제/재설치** → 로컬 DB 삭제 = 큐 소실 (현행, 정상 동작)

## 📐 일반 정책

### 에러 핸들링
- 앱 기존 일반 오류 처리 정책 따름 · 시트별 커스텀 불필요
- 일시적 오류 (네트워크/서버) → 토스트 알림 + 재시도 허용

### 데이터 리프레시
- ⚠️ **타 사용자 변경 실시간 싱크 없음** (WebSocket/폴링 없음)
- 시트 콘텐츠는 **화면 재진입(onResume) 시에만 새로고침**
- 📌 **온라인 시 항상 서버에서 최신 데이터를 가져옴** — 태그·코멘트·Box 모두 로컬 캐시가 아닌 서버 API를 호출하여 목록 구성. 로컬 DB에는 업로드 큐(pending) 아이템만 관리.
- 열려있는 동안 다른 사용자 수정 → 나갔다 돌아와야 반영
- 📤 **업로드 큐 상태는 실시간 반영** — Pending → Uploading → Succeeded/Failed 전이가 즉시 UI에 반영됨 (위 리프레시 정책과 별개)
