> 기획 스펙: SO-07 (Comment Sheet — List View), SO-08 (Comment Sheet — Detail View)

## 한줄 요약

> 스레드 기반 코멘트 시스템. 카테고리·상태(All/Unresolved/Resolved) 필터 리스트 + 스레드 디테일(답글·미디어·해결 처리). @멘션 지원 · 상세 방문 시 콘텐츠 로컬라이즈.

**기존 활용**: `SurveyOverviewCommentContent`, `SurveyOverviewCommentDetailView`, `SurveyOverviewCommentDelegate`, `MentionUseCase` + `MentionTextField`, 상태 필터 `DropdownMenu` 동작 중.

## 📋 리스트 뷰 (SO-07)

### 헤더
- "Comment" 레이블 + ➕ 추가 버튼 (온라인/오프라인 모두 활성)
- ➕ 탭 → 코멘트 생성: ① 도면에서 위치 지정 ② 전체 페이지 입력
- 📌 **카테고리 선택은 선택사항** — 카테고리 없이 생성 가능

### 카테고리 필터 바
- 태그와 동일한 가로 스크롤 칩 · 동일 카테고리 데이터셋
- 기본: 필터 없음

### 카운트 + 상태 필터
- `"{N} listings"` — 현재 뷰 코멘트 수
- 상태 필터 (DropdownMenu 3옵션):

| 옵션 | 표시 |
|------|------|
| **All** | 전체 |
| **Unresolved** | 미해결만 |
| **Resolved** | 해결됨만 |

### 코멘트 아이템

| 구성요소 | 내용 |
|---------|------|
| 헤드 행 | 프로필 아이콘 · 전체 이름 · 날짜 |
| 설명 | 코멘트 본문 (긴 경우 잘라냄) |
| 미디어 | 첨부 미디어 썸네일 그리드 |
| 카테고리 | 카테고리 레이블 |
| Thread 버튼 | 탭 → 디테일 뷰 |
| Resolve 버튼 | 탭 → 해결됨 표시 |

> 📌 **코멘트 아이템 전체 영역 탭** → 디테일 뷰 진입 (Thread 버튼뿐 아니라 아이템 어디든 탭) — ⚠️ 현재 구현은 Thread pill 버튼만 탭 가능, 전체 영역 탭 미구현

> 📅 **날짜 형식**: 오늘 → `Today HH:mm` / 7일 이내 → `요일 HH:mm` / 7일 초과 → `YYYY-MM-DD`

> 🔐 **Resolve 권한**: Collaborator, Surveyor, Team Admin, Site Manager, Super Admin (**Viewer 불가**)

### 위치 없는 코멘트 (블로커 해소)
- 모바일 P1 생성 시 위치 필수 / 웹 생성은 위치 없을 수 있음
- 📌 **정책**: 리스트에 정상 표시 · 캔버스 마커 없음 · 포커스 조용히 건너뜀
- ⚠️ `(0,0,0)`은 유효 좌표 → 위치 없음은 **null 또는 별도 플래그** 사용

### 수정·삭제 공통 불변식 (2026-06-10 추가)

> 📌 **포탈에 존재하는 코멘트·스레드 답글의 로컬 데이터 수정은 서버 API가 성공한 후에만 수행한다.** 서버 호출 실패 시 로컬 원본 보존 + 실패 안내. (태그도 동일 — PRD-03 참조)
> 큐 기반 편집 흐름에서는 큐 상태(Pending/Uploading/Failed)가 UI에 명시 표시되는 동안만 로컬 선기록이 허용되며, **확정(Completed) 전환은 서버 성공 시에만** 일어난다.

### 코멘트 삭제 처리 순서
- **태그와 동일 플로우** (PRD-03 참조): 온라인 클라우드 → 서버 삭제 후 로컬 삭제 / 오프라인 로컬 → 즉시 로컬 삭제

### 빈 상태 (Empty State)

| 상태 | 표시 |
|------|------|
| **온라인 + 코멘트 0개** | 기존 빈 상태 ("No Comments yet") |
| **오프라인 + 로컬 데이터 0개** | 온라인과 **동일한 기존 빈 상태** — 🚫 네트워크 없음 일러스트 사용 안 함 (2026-06-10 변경) |

### 오프라인 (2026-06-10 개정)

- **온라인 리스트 = 포탈 + 로컬 병합 풀 데이터** — 서버 코멘트 + 로컬 생성(큐) 아이템 모두 표시
- **오프라인 리스트 = 로컬 데이터만 표시**:
  - ① 로컬라이즈된 코멘트 — 상세 방문으로 콘텐츠 캐시된 클라우드 코멘트 (읽기 전용)
  - ② 로컬 생성(큐) 코멘트 — 업로드 큐 상태 인디케이터 포함
- 로컬 데이터 0개 → 기존 빈 상태 (네트워크 없음 일러스트 ✕)
- ➕ 활성 유지 → 오프라인 생성 → 업로드 큐
- 오프라인 CRUD: **생성 O · 읽기 O · 수정/삭제는 로컬 생성(Pending/Failed) 아이템만** — 클라우드·로컬라이즈 코멘트는 수정/삭제/Resolve 불가 (싱크 충돌 방지)
- 업로드 큐 상태는 **태그와 동일** (PRD-03 참조): Pending → Uploading → Failed → Succeeded

## 🔍 디테일 뷰 (SO-08)

### 진입점
- 코멘트 리스트에서 아이템/Thread 버튼 탭
- 캔버스에서 코멘트 아이콘 탭

### 레이아웃
- 리스트 + 피처 바 **오버레이** · 열린 시점 시트 크기

### 헤더 (sticky)
- 프로필 아이콘 + 전체 이름 + 날짜
- 점 세 개 메뉴: **Edit** / **Delete** → 삭제 시 리스트 복귀
- 📌 **본인이 작성한 코멘트에만 표시** (역할 무관). 스레드 답글도 동일 — 본인 답글만 Edit/Delete 가능
- 📌 **Edit 시 위치(도면 좌표) 변경 불가** — 콘텐츠(텍스트, 미디어, 카테고리)만 수정 가능
- ✕ 닫기 → 리스트 복귀 + 캔버스 아이콘 idle

### 본문

| 섹션 | 내용 |
|------|------|
| **설명** | 전체 코멘트 본문 (스크롤 가능) · HTML → `displayContent` plain text 변환 (멘션 span 추출 → `<p>` strip → `</p>`→`\n` → 잔여 태그 strip → 엔티티 unescape) |
| **미디어** | 첨부 썸네일 (aspect-fill) · **탭 → 무조건 프리뷰 페이지 이동** (이미지·비디오 동일) · 📌 **최대 5장** · 프리뷰 뷰어 기준 = Box 뷰어 (비디오 인라인 즉시 재생 — PRD-03 참조, 2026-06-11) |
| **카테고리** | 할당된 카테고리 레이블 |

> ⚠️ **서버 데이터 모델 주의**: 코멘트 첨부파일은 서버에서 `isMedia=false`로 내려옴 (서버가 코멘트 첨부를 파일로 취급). 미디어 표시 여부는 `isMedia` 필드가 아닌 **`attachType`** 필드(`"image"`, `"video"`)로 판단해야 함. 로컬 pending 아이템은 `attachType="media"`로 저장됨.

### 스레드 섹션

- **`"{N} Threads"`** — 자식 답글만 (루트 미포함)
- **Resolve 버튼** — Viewer 제외 모든 역할
- 스레드 아이템 (시간순 ↑):
  - 프로필 + 이름 + 날짜 + 점 세 개 (Edit/Delete)
  - 답글 텍스트 + 미디어 썸네일 스트립 · 📌 **최대 5장**

### 스레드 입력 바

| 버튼 | 동작 |
|------|------|
| 🖼️ 갤러리 | 디바이스 갤러리 → 미디어 선택 → 첨부 |
| ✏️ 텍스트 | 자유 입력 |
| ➤ Send | 콘텐츠 있을 때만 표시 → 답글 게시 → 입력 초기화 |

- 스레드와 함께 스크롤 (웹 벤치마킹 기반 변경 — spec-review #20)
- ✅ `LazyColumn` 내부 마지막 아이템으로 배치 — `Column > LazyColumn(weight(1f), imePadding) { ...threads, InputBar }` 구조

### 스레드 답글 편집 (Edit)

> 📌 스레드 답글의 점 세 개 메뉴 → **Edit** 탭 시 진입. 본인 작성 답글만 가능.

#### 편집 vs 등록 플로우 비교

| 항목 | 등록 | 편집 |
|------|------|------|
| 실행 조건 | 온라인/오프라인 모두 | **온라인 전제** — 오프라인 편집 불가 |
| 서버 호출 | 업로드 큐(Pending → Uploading → Completed) | **등록과 동일하게 업로드 큐 사용** |
| 큐 상태 UI | 표시 (Pending/Uploading/Failed 아이콘) | **등록과 동일 (Pending/Uploading/Failed/Completed)** |
| 완료 피드백 | 업로드 완료 Push Notification | **등록과 동일 (큐 완료 알림)** |
| 위치(도면 좌표) 변경 | N/A | **불가** — 콘텐츠(텍스트·미디어)만 수정 |

**편집에 큐 사용 근거**: 편집도 큐를 거치면 기존 큐 인프라(상태 UI, 알림, 구독, 실패 재시도)가 그대로 적용됨. 등록과 편집의 UX 일관성 유지. 오프라인에서 편집 시작 → 온라인 복구 후 자동 반영 가능.

#### 이미지 편집 UX

- 편집 중 이미지를 **추가**하면 → 첨부 썸네일이 **즉시** 입력 영역에 표시 (저장 전 미리보기)
- **저장 완료 후** → 스레드 아이템이 업데이트된 이미지와 함께 자연스럽게 렌더링되어야 함
- ⚠️ **레이아웃 급변 방지**: 저장 후 이미지가 지연 표시되거나 레이아웃이 갑자기 늘어나는 현상은 버그. 저장 직후 이미지 높이를 미리 확보(placeholder)하거나, 애니메이션(`animateContentSize` 등)으로 부드럽게 전환할 것.

#### 편집 완료 피드백

- ✅ 저장 성공 → 큐 완료 알림 (등록과 동일): 큐가 Completed 상태로 전환될 때 표준 큐 완료 알림 발송
- ❌ 저장 실패 → 큐 Failed 상태 + 편집 상태 유지 (입력 내용 소실 없음) — 큐 재시도 메커니즘 활용 가능
- 📌 **큐 완료 알림 사용**: 편집도 등록과 동일한 큐 플로우를 타므로 완료 피드백도 큐 완료 알림으로 통일 (사용자 결정 오버라이드, 2026-06-05)

#### 오프라인 편집 처리

- 오프라인 상태에서 Edit 탭 시 → "오프라인 상태에서는 편집할 수 없습니다" 안내 후 메뉴 닫기
- 편집 도중 네트워크 끊김 → 에러 토스트 + 편집 상태 유지 (변경 내용 보존)

### 콘텐츠 로컬라이즈 (2026-06-10 추가)
- 온라인에서 디테일 뷰 진입 시 해당 코멘트의 **포탈 콘텐츠(본문·미디어·카테고리·스레드)를 로컬에 캐시** = 로컬라이즈
- 로컬라이즈 데이터는 **로컬 생성(큐) 데이터와 별도 구분 저장** — 업로드 큐에 들어가면 안 됨 (PRD-07 용어 정의 참조)
- 온라인 상세 재방문 시 캐시 갱신

### 오프라인 (2026-06-10 개정)
- 로컬라이즈된 코멘트 → 디테일(스레드 포함) **읽기 가능** · Edit/Delete/Resolve 불가 (싱크 충돌 방지)
- 로컬라이즈 안 된 클라우드 코멘트 → 오프라인 리스트에 미표시 (디테일 접근 자체가 불가)
- 오프라인 스레드 답글 → 업로드 큐 + 낙관적 즉시 표시 (생성은 오프라인 허용)
- 로컬 생성(큐) 코멘트: Edit/Delete 가능 (Pending — 포탈 미존재, 싱크 영향 없음) / Failed는 Retry/Delete만
- Succeeded(포탈 업로드 완료) → 클라우드 데이터로 전환, 이후 수정/삭제는 온라인 전용
- 경쟁 조건: Save/Delete 확인 시점에 현재 상태 확인 후 라우팅
