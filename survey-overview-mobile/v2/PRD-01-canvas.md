# PRD-01: 캔버스 🗺️

> 기획 스펙: SO-01 (Floor Plan), SO-02 (Icon States), SO-03 (Interactions)

---

## 한줄 요약

> 도면 이미지 위에 포인트·태그·코멘트 아이콘을 오버레이하고, 줌/팬 + 아이콘 탭 인터랙션을 제공하는 메인 캔버스.

**기존 활용**: `FloorPlanView` + `FloorPlanState` (줌/팬, 핫스팟 렌더링) 동작 중 → 아이콘 비주얼 상태 확장 + 인터랙션 플로우 추가에 집중.

---

## 🖼️ 도면 렌더링

| 항목 | 동작 |
|------|------|
| **이미지 소스** | 전체 해상도 플랜 이미지 · 오프라인/미다운로드 시 썸네일 폴백 |
| **초기 줌** | fit-to-contain — 전체 도면이 캔버스 내 크롭 없이 표시, 즉시 렌더링 |
| **줌/팬** | 핀치 줌 min~8-10× · 싱글 핑거 드래그 패닝 |
| **경계 제한** | 도면이 뷰포트를 완전히 벗어나지 않도록 · 가장자리 48px 고정 마진 |
| **기존 코드** | `retryFindSurveyLocalImageBitmap()` (3회 재시도), `fitZoomLevel` pinch 최소 바운드, `offsetX/Y` coerceIn 클램핑 |

### 레이어 Z-ordering

```
도면 이미지 (최하위)
  → 오브젝트 아이콘 (포인트, 태그, 코멘트)
    → 3DW 버튼 (우상단)
    → Start Survey FAB (우하단)
```

## 📌 아이콘 렌더링

### 공통 규칙

- 🔒 **고정 화면-공간 크기** — 줌 레벨에 비례하지 않음
- 📍 도면 경계 밖 오브젝트도 정확한 위치에 렌더링
- ⬇️ 뾰족한 아이콘(태그·코멘트) 앵커 = **아이콘 바닥 팁**

### 상태별 비주얼·인터랙션 통합 매트릭스

| 아이콘 | 상태 | 비주얼 | 의미 | 탭 |
|--------|------|--------|------|-----|

| **태그** | Selected | ~120-130% 크기 | 현재 선택됨 | — |

| **코멘트** | 업로드 미완료 (Pending/Uploading/Failed) | **WHITEOpacity50 오버레이**, `alpha 1.0` | 사용자 생성, 아직 포탈에 업로드되지 않음 (재시도 대기 포함) | 오프라인 ❌ · 온라인 ✅ |
| **코멘트** | Selected | ~120-130% 크기 | 현재 선택됨 | — |

> **비주얼 규칙 요약** (2026-06-10 개정)
> - **WHITEOpacity50 오버레이(반투명) = 업로드 미완료(Pending/Uploading/Failed) 태그·코멘트 전용**. 반투명 흰색 원 오버레이 방식 (ColorMatrix 미사용)
> - **오프라인의 클라우드(로컬라이즈) 아이콘은 풀 컬러 유지** — 오버레이 없음, 탭 불가(비인터랙션)만 적용. 비주얼과 인터랙션 상태는 별개
> - 포인트는 오버레이 대신 별도 인디케이터(lock badge 등) 사용
> - **비활성화 (탭 불가)** = 오프라인 상태의 모든 클라우드 아이콘 + 오프라인 중 업로드 미완료 아이콘
> - 📌 캔버스 아이콘은 탭 불가지만, **콘텐츠 로컬라이즈된 태그/코멘트는 바텀 시트 리스트에서 읽기 가능** (PRD-03/04/07 참조, 2026-06-10). 로컬라이즈 아이콘의 캔버스 탭 허용 여부는 PRD-07 OQ#8

### 태그 아이콘 세부

- 기본 시스템: `i`/`!`/`?`/`✓` + 컬러
- 커스텀 아이콘: Space당 최대 5개, 동일 팁 프레임 내 렌더링
- 그레이스케일: `WHITEOpacity50` 반투명 흰색 원 오버레이 방식 (ColorMatrix 미사용) — 새 에셋 불필요

### 코멘트 아이콘 세부

- 아래 방향 팁 + **생성자 프로필 사진** (오프라인: 캐시 → 없으면 기본 아바타)

## 👆 인터랙션

### 포인트 탭

| 케이스 | 동작 |
|--------|------|
| ☁️ **클라우드 포인트** (온라인) | 컨펌 → 모바일 3DWorkspace 인앱 브라우저 |
| ☁️ **클라우드 포인트** (오프라인) | 비인터랙션 |
| 📱 **로컬 Downloaded** | 컨펌 → 360° 이미지 뷰어 |
| 📱 **로컬 Not downloaded** | 컨펌 → 카메라 연결 → 임포트 → 뷰어 (카메라 미연결 시 연결 플로우 선행) |

> 💡 클라우드 포인트에 컨펌 다이얼로그가 있는 이유: 3DW는 전체 페이지 내비게이션이라 실수 탭 복구 비용이 큼

### 태그·코멘트 아이콘 탭

| 상태 | 동작 |
|------|------|
| **온라인 (클라우드)** | 바텀 시트 → 디테일 뷰 (mid 크기) · 아이콘 selected · 캔버스 패닝 |
| **온라인 (업로드 대기)** | 바텀 시트 → 디테일 뷰 (로컬 데이터 기반) · 아이콘 selected · 캔버스 패닝 |
| **오프라인** | 비인터랙션 |

### 캔버스 빈 영역 탭

- 빈 영역 **탭** → 선택된 아이콘 해제 + 디테일 뷰 닫힘 (idle 복귀)
- 드래그/팬 **무빙** → 선택 상태 및 디테일 뷰 유지

### 캔버스 포커스 동작

> 아이콘 선택 시 캔버스가 **non-overlaid 영역 중앙**으로 부드럽게 패닝 (~250-300ms)

- Non-overlaid 영역 = mid-height 시트 상단 위 직사각형
- ~~이미 영역 내에 있으면 패닝 생략~~ (현재 미구현 — 무조건 패닝)
- 다른 오브젝트 선택 / 디테일 닫기 / 빈 영역 탭 → 이전 아이콘 idle 복귀
- 드래그/팬 시에는 선택 상태 유지

### 탭 타겟

- 전체 아이콘 바운딩 박스 = 탭 가능 영역 (팁~전체 높이 확장)
- `doubleClickCheck()` 1.5초 재클릭 방지 구현 완료

### 아이콘 겹침 시 탭 우선순위

- **최신 생성순(Z-order)** — 최신 아이콘이 위에 렌더링
- 완전히 겹칠 경우 최신 아이콘이 탭됨
