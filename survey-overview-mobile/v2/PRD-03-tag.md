## 📋 리스트 뷰 (SO-05)

### 헤더

| 위치 | 구성 |
|------|------|
| **루트 레벨** | "Tag" 레이블 + ➕ 추가 버튼 |
| **폴더 내부** | ← 뒤로 + 폴더명 (최대 2줄) + ➕ 추가 버튼 |

**➕ 버튼 탭** → 옵션 피커: `Tag` \| `Folder`
- Tag → 태그 생성 (도면 위치 지정 → 태그 입력) — **현재 보고 있는 뎁스에 생성**, 폴더 선택 UI 없음
- Folder → 인라인 폴더 생성
- 📌 2-depth(최하위 폴더)에서는 "Add Folder" 미표시, "Add Tag"만 표시 — ⚠️ 현재 미구현 (`TagAddTypeBottomSheet`이 depth 무관하게 항상 두 옵션 모두 표시)
- 📌 **카테고리 선택은 선택사항** — 카테고리 없이 생성 가능

### 카테고리 필터 바

- 가로 스크롤 카테고리 칩 · 기본 "All" = 필터 없음
- 칩 탭 → 해당 카테고리 태그만 표시
- 📌 **시스템(기본) 카테고리 정책** (2026-06-11 정밀화):
  - 서버에는 isDefault 시스템 카테고리(`media`/`attachment` 등)가 **실존**하며, 동기화하여 **필터 칩에 노출**한다 (Tags/Comments/Box 탭 공통)
  - 생성·수정 화면의 Media/Attachment 자동 칩은 표시용이며, 저장 시 시스템 카테고리는 **서버로 전송하지 않는다** (서버가 자동 부여/관리 — 카테고리 API diff 및 PUT 대상 아님)
- 📌 **폴더는 필터와 무관하게 항상 표시**
- 📌 **폴더 진입 후에도 카테고리 필터 유지** — 폴더 내부 태그도 필터 적용됨
- 기존: `OverviewCategoryChipRow` 공유 컴포넌트

### 카운트 + 정렬

- `"{N} listings"` — 현재 뷰 총 아이템 수 (필터 반영)
- 📌 **카테고리 필터 + 정렬은 동시 적용** — 예: "Category A" 필터 + "Name A-Z" → Category A인 태그만 알파벳순 표시

| 정렬 옵션 | 설명 |
|-----------|------|
| **Last updated** | 마지막 업데이트 ↓ |
| **Created** | 생성일 ↓ |
| **Name (A-Z)** | 알파벳 ↑ |
| **Custom Order** | 웹 정의 순서, 모바일에서는 읽기 전용 (드래그 순서 변경은 P2) |

### 폴더 아이템

- 📁 폴더 아이콘 + 타이틀 + **내부 아이템 수 배지**
- 📌 **카테고리 필터 적용 시 배지 = 필터된 아이템 수** — 해당 태그 없으면 0 표시
- 탭 → 폴더 안으로 진입
- 점 세 개 메뉴:
  - **Rename** — 인라인 편집 (기존 텍스트 선택 + 키보드)
  - **Delete** — 컨펌 → 폴더만 삭제, **자식 태그는 루트로 이동** (서버가 폴더 ID 제거)

### 태그 아이템

- 🏷️ 태그 아이콘 (타입/색상 또는 커스텀) + 타이틀 + 마지막 업데이트 날짜
- 📅 **날짜 형식**: `yyyy-MM-dd` (예: `2025-03-22`)
- 탭 → ① 시트 mid 크기 ② 디테일 뷰 ③ 캔버스 아이콘 selected ④ 캔버스 포커스 패닝
- 점 세 개 메뉴: **Edit** / **Delete** (컨펌) — 📌 역할 기반 권한 (Viewer 제외 전원 가능, 소유권 체크 없음)
- 📌 **Edit 시 폴더 변경 불가** — 다른 폴더로 이동하려면 삭제 후 재생성
- 📌 **Edit 시 위치(도면 좌표) 변경 불가** — 콘텐츠(텍스트, **아이콘**, 미디어, 첨부, 카테고리)만 수정 가능
- 📌 **아이콘 미변경 시 기존 아이콘 유지** (2026-06-10 명시 — 편집 진입 시 기존 아이콘이 바인딩되어야 하며, 설명만 수정해도 아이콘이 기본값으로 리셋되면 버그)

### 태그 수정 처리 순서 (2026-06-10 추가, 웹 실동작 기준으로 개정)

> 📌 **공통 불변식: 로컬 데이터 수정은 서버 API가 전부 성공한 후에만 수행한다.** 서버 호출이 하나라도 실패하면 로컬 원본을 보존하고 실패를 안내한다. **첨부파일 변경(추가/삭제)도 동일** — 첨부 API 성공 후에만 로컬 첨부 캐시 갱신. (태그·코멘트·스레드 답글 공통 — PRD-04/PRD-07 참조)

**웹 포탈과 동일한 수정 프로세스를 따른다** (실제 웹 페이로드 분석 기준, 2026-06-10):

| 단계 | API | 내용 |
|------|-----|------|
| ① 본문 수정 | `PUT /v2/tag/tags/{id}?teamId={teamId}` | **teamId 쿼리 필수.** 바디는 태그 **전체 스냅샷 echo** + 변경 필드 — position·icon·tourId·sceneId·type·content·title·media·attachs·categories(객체 배열 `{id,text,tagId}`) 포함. 부분 필드만 보내면 400 |
| ② 첨부 삭제 | `DELETE /v2/att/attachs/bulk?teamId=` | 바디 `{"ids": [attachId...]}` — 삭제할 첨부의 서버 id 목록 |
| ③ 첨부 추가 | `POST /v2/att/attachs/bulk?teamId=` | multipart: `relatedItemId`(태그 id), `relatedItemType="tag"`, `files`(바이너리), `items`(JSON: fileName/originalFileName/isMedia) |

- 이미지 **교체** = ① PUT + ② 기존 미디어 DELETE + ③ 새 이미지 POST (3콜)
- 카테고리 변경은 카테고리 API diff(add/remove)로 별도 적용
- 모든 호출 성공 → 그때만 로컬 캐시 갱신 (internalTagId 보존 upsert)

| 상태 | 처리 |
|------|------|
| **포탈 태그 (tagId 보유)** | 위 프로세스로 **update-in-place**. 🚫 신규 태그 생성 절대 금지 |
| **수정 복귀 시 화면 갱신** | 수정 저장 후 상세 뷰로 복귀하면 **수정 내용이 즉시 반영**되어야 함 — 리스트·상세·캔버스 모두 (2026-06-11 추가, 미갱신은 버그) |
| **로컬 생성 태그 (업로드 전)** | 로컬에서 교체 (아직 포탈에 없으므로 서버 선행 조건 없음) |
| **수정 실패 (서버 오류)** | 원본 보존 — 로컬 데이터 변경 없음, 에러 안내 |

> ⚠️ 발견 버그 이력 (2026-06-10): ① 수정이 "로컬 삭제 후 신규 생성"으로 동작해 포탈에 태그 중복 생성 → update-in-place로 수정. ② PUT을 부분 바디로 보내 400 → 전체 스냅샷 echo + teamId 쿼리로 수정.

### 태그 삭제 처리 순서

| 상태 | 처리 |
|------|------|
| **온라인 + 클라우드 태그 (로컬라이즈 됨)** | 서버 삭제 → 성공 후 로컬(로컬라이즈 캐시) 삭제 (직렬) |
| **온라인 + 클라우드 태그 (로컬라이즈 안 됨)** | 서버만 삭제 |
| **오프라인 + 로컬 생성 태그 (Pending/Failed — 포탈 미업로드)** | 로컬만 삭제 (큐에서 제거) — 싱크 영향 없음 |
| **오프라인 + 클라우드/로컬라이즈 태그** | 🚫 **삭제 불가** — 메뉴 숨김/비활성 (싱크 충돌 방지, 2026-06-10 결정) |

> 코멘트도 동일한 삭제 플로우 적용 (PRD-04 참조)

### 빈 상태 (Empty State)

| 상태 | 표시 |
|------|------|
| **온라인 + 태그 0개** | 텍스트만: "No Tags yet" / "Start by adding a new tag!" (아이콘 없음) |
| **오프라인 + 로컬 데이터 0개** | 온라인과 **동일한 기존 빈 상태** ("No Tags yet") — 🚫 네트워크 없음 일러스트 사용 안 함 (2026-06-10 변경) |

> 기존 구현: `SurveyOverviewTagContent` → `BeamoEmptyView`

### 오프라인 (2026-06-10 개정)

- **온라인 리스트 = 포탈 + 로컬 병합 풀 데이터** — 서버 태그 + 로컬 생성(큐) 아이템 모두 표시
- **오프라인 리스트 = 로컬 데이터만 표시**:
  - ① 로컬라이즈된 태그 — 상세 방문으로 콘텐츠 캐시된 클라우드 태그 (읽기 전용)
  - ② 로컬 생성(큐) 태그 — 업로드 큐 상태 인디케이터 포함
- 로컬 데이터 0개 → 기존 빈 상태 "No Tags yet" (네트워크 없음 일러스트 ✕)
- ➕ 버튼 **활성 유지** → 오프라인 태그/폴더 생성 → 업로드 큐
- 오프라인 CRUD: **생성 O · 읽기 O · 수정/삭제는 로컬 생성(Pending/Failed) 아이템만** — 클라우드·로컬라이즈 태그는 수정/삭제 불가 (싱크 충돌 방지)
- 📌 **네트워크 복구 시 큐 자동 처리** — FIFO 순차 업로드 시작 (PRD-07 참조)

## 🔍 디테일 뷰 (SO-06)

### 진입점
- 태그 리스트에서 아이템 탭
- 캔버스에서 태그 아이콘 탭

### 레이아웃
- 리스트 뷰 + 피처 바를 **오버레이**
- 시트 크기: 열린 시점의 크기 따름

### 헤더 (sticky)
- 태그 타이틀 (최대 2줄 말줄임표)
- 점 세 개 메뉴: **Edit** / **Delete** → 삭제 시 리스트 복귀
- ✕ 닫기 → 리스트 복귀 + 캔버스 아이콘 idle

### 본문
| 섹션 | 내용 |
|------|------|
| **설명** | 태그 설명 텍스트 (스크롤 가능, 길이 제한 현행 유지 — 서버 제한 기준) |
| **카테고리** | 할당된 카테고리 칩/레이블 |
| **미디어** | 첨부 미디어 그리드 (aspect-fill 크롭) · **탭 → 무조건 프리뷰 페이지 이동** (이미지·비디오 동일 — 외부 플레이어/인라인 즉시 실행 금지, 2026-06-11 명시) · 📌 **최대 1장** |
| **첨부파일** | 파일 첨부 · 📌 **최대 5개** |

> 📌 **프리뷰 페이지 미디어 뷰어 기준 = Box 뷰어** (2026-06-11 추가): 이미지 = 줌/팬 제스처, **비디오 = 페이지 내 즉시 재생(인라인 플레이어 + 컨트롤러)** — 외부 앱 호출 없이 프리뷰 안에서 재생. 태그·코멘트·Box 모든 프리뷰가 동일 뷰어 UX를 따른다.

> 📌 **서버 측 첨부 제한** (2026-06-10 추가): 파일당 최대 **30MB**, 일괄 업로드 최대 **10개** (`ATTACHS_FILE_SIZE_LIMIT`). UI 제한(미디어 1장·첨부 5개)이 더 좁으므로 평시엔 UI 제한이 우선하나, 클라이언트는 업로드 전 30MB 초과 파일을 사전 검증해 거부한다.

### 콘텐츠 로컬라이즈 (2026-06-10 추가)
- 온라인에서 디테일 뷰 진입 시 해당 태그의 **포탈 콘텐츠(설명·카테고리·미디어)를 로컬에 캐시** = 로컬라이즈
- 로컬라이즈 데이터는 **로컬 생성(큐) 데이터와 별도 구분 저장** — 업로드 큐에 들어가면 안 됨 (PRD-07 용어 정의 참조)
- 온라인 상세 재방문 시 캐시 갱신

### 오프라인
- 로컬 생성(큐) 태그 → 로컬 데이터 표시 · Pending/Failed 상태면 수정/삭제 가능
- 로컬라이즈된 태그 → 디테일 **읽기 가능** · 수정/삭제 불가 (점 세 개 메뉴 숨김/회색 — 싱크 충돌 방지)
- 로컬라이즈 안 된 클라우드 태그 → 오프라인 리스트에 미표시 (디테일 접근 자체가 불가)
