# PRD-03: 태그 🏷️

> 기획 스펙: SO-05 (Tag Sheet — List View), SO-06 (Tag Sheet — Detail View)
> 역할별 CRUD 권한: `_shared/roles.md` 참조

---

## 한줄 요약

> 폴더 계층 태그 리스트 + 개별 태그 디테일 뷰. 카테고리 필터·4종 정렬·full CRUD · 오프라인 생성→업로드 큐.

**기존 활용**: `SurveyOverviewTagContent`, `SurveyOverviewTagDetailContent`, `SurveyOverviewTagDelegate`, `TagAddTypeBottomSheet`, 인라인 폴더 생성/리네임 동작 중.

## 📋 리스트 뷰 (SO-05)

### 헤더

| 위치 | 구성 |
|------|------|
| **루트 레벨** | "Tag" 레이블 + ➕ 추가 버튼 |
| **폴더 내부** | ← 뒤로 + 폴더명 (최대 2줄) + ➕ 추가 버튼 |

**➕ 버튼 탭** → 옵션 피커: `Tag` \| `Folder`
- Tag → 태그 생성 (도면 위치 지정 → 태그 입력) — **현재 보고 있는 뎁스에 생성**, 폴더 선택 UI 없음
- Folder → 인라인 폴더 생성
- 📌 2-depth(최하위 폴더)에서는 "Add Folder" 미표시, "Add Tag"만 표시 — ⚠️ 현재 미구현 (`TagAddTypeBottomSheet`이 depth 무관하게 항상 두 옵션 모두 표시)
- 📌 **카테고리 선택은 선택사항** — 카테고리 없이 생성 가능

### 카테고리 필터 바

- 가로 스크롤 카테고리 칩 · 기본 "All" = 필터 없음
- 칩 탭 → 해당 카테고리 태그만 표시
- 📌 **폴더는 필터와 무관하게 항상 표시**
- 📌 **폴더 진입 후에도 카테고리 필터 유지** — 폴더 내부 태그도 필터 적용됨
- 기존: `OverviewCategoryChipRow` 공유 컴포넌트

### 카운트 + 정렬

- `"{N} listings"` — 현재 뷰 총 아이템 수 (필터 반영)
- 📌 **카테고리 필터 + 정렬은 동시 적용** — 예: "Category A" 필터 + "Name A-Z" → Category A인 태그만 알파벳순 표시

| 정렬 옵션 | 설명 |
|-----------|------|
| **Last updated** | 마지막 업데이트 ↓ |
| **Created** | 생성일 ↓ |
| **Name (A-Z)** | 알파벳 ↑ |
| **Custom Order** | 웹 정의 순서, 모바일에서는 읽기 전용 (드래그 순서 변경은 P2) |

### 폴더 아이템

- 📁 폴더 아이콘 + 타이틀 + **내부 아이템 수 배지**
- 📌 **카테고리 필터 적용 시 배지 = 필터된 아이템 수** — 해당 태그 없으면 0 표시
- 탭 → 폴더 안으로 진입
- 점 세 개 메뉴:
  - **Rename** — 인라인 편집 (기존 텍스트 선택 + 키보드)
  - **Delete** — 컨펌 → 폴더만 삭제, **자식 태그는 루트로 이동** (서버가 폴더 ID 제거)

### 태그 아이템

- 🏷️ 태그 아이콘 (타입/색상 또는 커스텀) + 타이틀 + 마지막 업데이트 날짜
- 📅 **날짜 형식**: `yyyy-MM-dd` (예: `2025-03-22`)
- 탭 → ① 시트 mid 크기 ② 디테일 뷰 ③ 캔버스 아이콘 selected ④ 캔버스 포커스 패닝
- 점 세 개 메뉴: **Edit** / **Delete** (컨펌) — 📌 역할 기반 권한 (Viewer 제외 전원 가능, 소유권 체크 없음)
- 📌 **Edit 시 폴더 변경 불가** — 다른 폴더로 이동하려면 삭제 후 재생성
- 📌 **Edit 시 위치(도면 좌표) 변경 불가** — 콘텐츠(텍스트, 미디어, 카테고리)만 수정 가능

### 태그 삭제 처리 순서

| 상태 | 처리 |
|------|------|
| **온라인 + 클라우드 태그 (로컬라이즈 됨)** | 서버 삭제 → 성공 후 로컬 삭제 (직렬) |
| **온라인 + 클라우드 태그 (로컬라이즈 안 됨)** | 서버만 삭제 |
| **오프라인 + 로컬 생성 태그** | 로컬만 삭제 (큐에서 제거) |

> 코멘트도 동일한 삭제 플로우 적용 (PRD-04 참조)

### 업로드 큐 상태

| 상태 | UI (점 세 개 메뉴 위치) | 비고 |
|------|----------------------|------|
| ⏳ **Pending** | 클라우드-싱크 아이콘 | 큐 대기 |
| 🔄 **Uploading** | 애니메이션 프로그레스 | 업로드 중 |
| ❌ **Failed** | `Retry` + `Delete` 버튼 | 재시도/삭제 |
| ✅ **Succeeded** | 정상 점 세 개 메뉴 복원 | 클라우드 전환 |

### 빈 상태 (Empty State)

| 상태 | 표시 |
|------|------|
| **온라인 + 태그 0개** | 텍스트만: "No Tags yet" / "Start by adding a new tag!" (아이콘 없음) |
| **오프라인 + 클라우드 데이터 없음** | 🚫 네트워크 없음 일러스트: "No internet connection" |

> 기존 구현: `SurveyOverviewTagContent` → `BeamoEmptyView`

### 오프라인

- 오프라인 시 **클라우드 태그 리스트 미표시** → 네트워크 없음 일러스트로 대체
- ➕ 버튼 **활성 유지** → 오프라인 태그/폴더 생성 → 업로드 큐
- 로컬 생성(큐) 아이템은 오프라인에서도 리스트에 표시 (업로드 큐 상태 인디케이터 포함)
- 📌 **네트워크 복구 시 큐 자동 처리** — FIFO 순차 업로드 시작 (PRD-07 참조)

## 🔍 디테일 뷰 (SO-06)

### 진입점
- 태그 리스트에서 아이템 탭
- 캔버스에서 태그 아이콘 탭

### 레이아웃
- 리스트 뷰 + 피처 바를 **오버레이**
- 시트 크기: 열린 시점의 크기 따름

### 헤더 (sticky)
- 태그 타이틀 (최대 2줄 말줄임표)
- 점 세 개 메뉴: **Edit** / **Delete** → 삭제 시 리스트 복귀
- ✕ 닫기 → 리스트 복귀 + 캔버스 아이콘 idle

### 본문
| 섹션 | 내용 |
|------|------|
| **설명** | 태그 설명 텍스트 (스크롤 가능, 길이 제한 현행 유지 — 서버 제한 기준) |
| **카테고리** | 할당된 카테고리 칩/레이블 |
| **미디어** | 첨부 미디어 그리드 (aspect-fill 크롭) · 탭 → 전체 뷰어 |

### 오프라인
- 로컬 큐된 태그 → 로컬 데이터 표시
- 클라우드 태그 → 온라인에서만 편집/삭제 (오프라인 시 메뉴 숨김/회색)
