### Q. 스레드 답글 편집 — 이미지 레이아웃 급변, 큐 상태 UI, 편집 완료 알림 처리 방향

| 항목 | 내용 |
|------|------|
| **작성자** | PM (wooseung.kim@3i.ai) |
| **작성일** | 2026-06-05 |
| **해결 여부** | ✅ 해결 (PM 판단 완료, PRD-04 반영) |
| **질문** | 스레드 답글 편집 기능에서 3가지 이슈가 보고됨: ① 이미지 추가 후 저장 시 레이아웃 급변, ② 편집 시 큐 상태 UI 표시 여부, ③ 편집 완료 Push Notification 필요 여부 |

#### A.

| 항목 | 내용 |
|------|------|
| **답변자** | PM (wooseung.kim@3i.ai) |
| **답변일** | 2026-06-05 |
| **답변** | 아래 세 항목 각각 PM 판단으로 정리 |

**이슈 ① 이미지 추가 후 레이아웃 급변**

버그로 규정. 편집 저장 완료 후 이미지가 지연 표시되거나 레이아웃이 갑자기 늘어나는 현상은 허용 불가. placeholder를 미리 확보하거나 `animateContentSize` 등으로 부드럽게 전환하도록 구현 명세에 추가.

**이슈 ② 편집 시 큐 상태 UI 표시 여부**

편집은 큐 미사용, 직접 서버 API 호출이 맞다. 근거: 등록은 오프라인 지원을 위해 큐가 필요하나 편집은 온라인 전제 작업이며, 사용자가 화면을 직접 보고 있는 상태에서 큐를 거치면 완료 시점이 불명확해져 UX가 나빠진다. 편집 중 로딩 인디케이터(스피너 등)는 표시하되 큐 상태 UI(Pending/Uploading 아이콘)는 불필요.

**이슈 ③ 편집 완료 Push Notification 필요 여부**

불필요. 등록 알림은 사용자가 앱 백그라운드에 있는 상황을 가정하지만, 편집은 사용자가 해당 화면을 직접 보고 있는 온라인 작업이다. 인앱 토스트/스낵바(`"Thread reply updated"`)로 충분하며 Push Notification은 발송하지 않는다.
